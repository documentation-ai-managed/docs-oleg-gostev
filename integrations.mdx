## Webhook Configurations

Set up webhooks to receive real-time events from Oleg Gostev. Webhooks trigger on events like project updates or user actions.

### Create a Webhook

Follow these steps to configure a webhook:

<Steps>
  <Step title="Generate Webhook URL" icon="link">
    In the dashboard, navigate to Webhooks > New Webhook.
  </Step>
  <Step title="Add Endpoint" icon="globe">
    Enter your endpoint URL, such as `https://your-webhook-url.com/webhook`.
  </Step>
  <Step title="Select Events" icon="bell">
    Choose events like `project.updated` or `user.created`.
  </Step>
  <Step title="Save and Test" icon="play">
    Save the webhook and send a test payload.
  </Step>
</Steps>

Example payload received:

````json
{
  "event": "project.updated",
  "data": {
    "id": "proj_123",
    "name": "My Project",
    "updated_at": "2024-01-15T10:30:00Z"
  }
}
````

<Expandable title="Advanced Webhook Filtering" default-open="false">
  Use filters to subscribe only to specific conditions, like projects in a certain status.
</Expandable>

## API Access for Custom Integrations

Build custom integrations using the Oleg Gostev API at `https://api.example.com`. Authenticate with your API key.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Authorization: Bearer YOUR_API_KEY`
</ParamField>

<ParamField query="project_id" param-type="string" required="false">
  Filter by project ID.
</ParamField>

<CodeGroup tabs="JavaScript,cURL">
  ````javascript
  const response = await fetch('https://api.example.com/v1/projects', {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  });
  const projects = await response.json();
  console.log(projects);
  ````
  ````bash
  curl -H "Authorization: Bearer YOUR_API_KEY" \
       https://api.example.com/v1/projects
  ````
</CodeGroup>
